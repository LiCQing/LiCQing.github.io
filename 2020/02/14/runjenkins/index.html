一、Jenkins介绍

来自官网的介绍：

作为一个可扩展的自动化服务器，Jenkins 可以用作简单的 CI 服务器，或者变成任何项目的持续交付中心。

我的理解：

可以将代码从SVN或GIT仓库拉取过来进行自动编译与发布，所以可以成为一个交付中心，也可以作为一个集成测试环境。

2.3 总结

1.安装流程如下：获取压缩包-》解压指定文件位置-》配置环境变量-》刷新环境变量

2.还有一种安装方法是通过yum源来安装，相当于在windows下的自动安装，如果不熟悉不太方便掌握文件位置

3.本篇基于百度云服务器纯净的Centos版本，为后续Jenkins安装部署做铺垫

Docker 安装

uname -r 查看内核

前置 sudo yum install -y yum-utils device-mapper-persistent-data lvm2
input // already installed and latest version
添加国内镜像源 sudo yum-config-manager –add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
更新缓存 sudo yum makecache fast
安装docker-ce sudo yum -y install docker-ce
启动dokcer sudo systemctl start docker
测试运行 hello-world： docker run hello-world #很慢

修改镜像
sudo mkdir -p /etc/docker
sudo vim /etc/docker/daemon.json
{
“registry-mirrors”:[“http://hub-mirror.c.163.com"]
}
systemctl restart docker
一些加速站点

配置Docker工作空间
mkdir /home/work/demo
cd /home/work/demo
mkdir backup
vim Dockerfile

安装Jenkins

官网 https://jenkins.io/zh/

安装步骤
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat-stable/jenkins.repo
sudo rpm –import https://pkg.jenkins.io/redhat-stable/jenkins.io.key
yum install jenkins

service jenkins start
//失败
vim /etc/init.d/jenkins
vim /etc/sysconfig/jenkins

重启
systemctl daemon-reload
service jenkins start

//成功
[root@instance-7k1d5qb8 ~]# service jenkins start
Starting jenkins (via systemctl): [ OK ]

//访问失败
sudo yum install libgcc.i686 –setopt=protected_multilib=false
